# Security Policy

## Supported Versions

At this stage, `project-history` is pre-1.0. Only the **latest released version**
receives security updates.

| Version  | Supported          |
|---------|--------------------|
| 0.1.x   | ✅                  |
| < 0.1   | ❌ (unsupported)    |

As the project stabilizes and reaches 1.0, this table will be updated with a
more formal support window.

---

## Reporting a Vulnerability

If you believe you have found a security issue in `project-history` (for example,
RCE via CLI input, DB corruption, or data exposure), please report it privately.

**Do not** create a public GitHub issue with details of the vulnerability.

### How to report

1. Email the maintainer at:

   > **security contact:** `jessejthrift@includeBrake.com`

2. Include, if possible:
   - A short description of the issue  
   - Steps to reproduce  
   - Any proof-of-concept code or CLI commands  
   - Your environment (OS, Python version, `ph` version)

### What to expect

- **Acknowledgement:** within **72 hours** of your report.
- **Initial assessment:** within **7 days**, including whether the issue is
  accepted as a vulnerability and what the next steps are.
- **Fix & release:** for accepted issues, a fix will be developed and released
  as soon as reasonably possible, with credit given i
